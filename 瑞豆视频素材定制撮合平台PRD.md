## 1. 项目背景概述

### 1.1 产品定位
**瑞豆（Relidou）** 是一个商业广告视频素材定制撮合服务平台，连接广告主（甲方）和视频创作者，提供从需求发布、任务匹配、视频制作到验收交付的全流程服务。

### 1.2 目标用户
| 用户类型 | 角色 | 核心需求 |
|---------|------|---------|
| **广告主** | 甲方 | 发布视频素材需求，寻找优质创作者，快速获得高质量视频 |
| **创作者** | 乙方 | 浏览任务，承接订单，上传视频作品，获得收入 |
| **管理员** | 运营 | 审核任务、管理用户、处理投诉、平台运营 |

### 1.3 支持的平台
- 抖音、快手、小红书、视频号、B站、腾讯广告

### 1.4 技术架构
- **前端**：React 19.2 + TypeScript + Vite
- **路由**：React Router DOM 7.x (HashRouter)
- **UI组件**：Radix UI + Tailwind CSS
- **状态管理**：React Context API

#### 登录后跳转逻辑
**选择创作者**：
1. 判断手机号是否已注册：
   - **未注册** → 进入个人信息补充页 (/creator-profile-setup)
     - 可选择"跳过" → 进入任务大厅 (/orders)
     - 填写完成后 → 进入任务大厅 (/orders)
   - **已注册** → 直接进入任务大厅 (/orders)

**选择广告主**：
1. 判断手机号是否已注册：
   - **未注册** → 进入广告主信息认证页 (/client-verification)
     - 提交信息 → 等待审核
   - **已注册** → 判断审核状态：
     - **审核通过** → 直接进入广告主工作台 (/client-workspace)
     - **审核中** → 进入等待审核页 (/pending-review)
     - **审核驳回** → 显示驳回原因，可重新提交认证

## 5. 交互逻辑与跳转关系

### 5.1 核心业务流程

#### 流程0：登录注册流程（未登录用户）

```
【触发条件】
未登录用户点击导航栏「登录/注册」按钮

【角色选择】
1. 进入登录页 /login
   ↓
2. 选择用户角色：
   - 创作者：承接任务，上传视频作品
   - 广告主：发布视频需求，寻找创作者
   ↓
3. 输入手机号 + 验证码
   - 手机号：11位，格式1[3-9]\d{9}
   - 验证码：6位数字（测试码：123456）
   - "获取验证码"按钮：60秒倒计时
   ↓
4. 点击"登录/注册"按钮
   ↓
5. 系统判断手机号是否已注册

【创作者分支】
若未注册：
  → 进入个人信息补充页 /creator-profile-setup
  → 可选择"暂时跳过"或"完成"
  → 跳过后进入任务大厅 /orders（可浏览，接单时再次提示）
  → 完善后进入任务大厅 /orders

若已注册：
  → 直接进入任务大厅 /orders

【广告主分支】
若未注册：
  → 进入广告主认证页 /client-verification
  → 填写企业/个人信息
  → 提交认证 → 等待审核页 /pending-review
  → 审核1-3个工作日
  → 审核通过后可进入工作台

若已注册：
  → 判断认证状态：
    - 审核通过 → 进入广告主工作台 /client-workspace
    - 审核中 → 进入等待审核页 /pending-review
    - 审核驳回 → 显示驳回原因，可重新提交认证
```

**关键逻辑说明**：
1. **创作者注册流程轻量化**：
   - 首次登录可选择跳过个人信息填写
   - 跳过后仍可浏览任务大厅
   - 接单时再次提示完善信息（但不强制）

2. **广告主注册需要认证**：
   - 必须完成企业/个人认证
   - 认证通过后才能使用发布任务等功能
   - 审核周期1-3个工作日

3. **验证码设计**：
   - 60秒倒计时防重复发送
   - 测试环境固定验证码：123456

```
6. 查看新的投标（状态：预审已通过）
   ↓
7. 广告主操作选项：
   - "确认通过"：扣除尾款，交付无水印视频+授权书
   - "修改"：发送修改意见（最多500字）
   - "订单冻结"：处理纠纷
   - "查看创作者"：跳转到创作者主页
```

### 5.2 状态流转图

#### 订单状态流转（广告主视角）
```
编辑中 → AI预审 → 运营审核
             ↓        ↓
          继续修改   审核通过 ↓
                      ↓       ↓
                   审核驳回 ← 审核中 → 进行中
                      ↑        ↓
                      └── 已下架
                              ↓
                           已结束
```

#### 投标状态流转（创作者视角）
```
待上传视频 → 上传视频 → 待审核 → (修改反馈) → 待审核
                          ↓
                      待交付 → (广告主修改) → 待交付
                          ↓
                      已完成

终止流程（不可逆）：
待上传视频 → (超时未上传) → 自动取消
待审核 → (审核拒绝) → 已拒绝 ❌ 该创作者不能再对此订单接单
待交付 → (审核拒绝) → 已拒绝 ❌ 该创作者不能再对此订单接单

特殊情况：订单冻结（任何状态都可能）
```

**重要说明**：
- 创作者对同一个订单的投标机会只有**一次**
- 一旦投标被拒绝（status = rejected），该创作者将无法再次对该订单进行接单
- 这是平台的质量控制机制，确保创作者认真对待每一次投标
- 被拒绝的创作者可以在创作者工作台查看拒绝原因，但无法重新投标

### 6.3 业务规则约束

**0. 创作者个人信息完善要求 ⭐ 重要**

**2. 创作者投标机会限制 ⭐ 重要**
- 创作者对同一个订单的投标机会只有**一次**

- 如果存在被拒绝的记录，点击"立即抢单"时提示："您曾对该订单投标被拒绝，无法再次接单"

### 6.4 前置条件与后置条件

#### 广告主发布订单
**前置条件**：已登录且角色为广告主
**后置条件**：订单状态为"审核中"，等待运营审核

#### 创作者接单
**前置条件**：
- 已登录且角色为创作者
- **个人信息已完善**（⭐ 重要：`isProfileComplete = true`）
- 当前接单数量 < 3
- 订单状态为"审核通过"且未过期
- 该创作者对该订单**未被拒绝过**（⭐ 重要：检查投标表中是否存在 `orderId + creatorPhone` 组合且 `isRejected = true` 的记录）

**后置条件**：
- 创建投标记录（status = pending，isRejected = false）
- 跳转到创作者工作台

**不满足前置条件的处理**：
- 个人信息未完善：提示"接单前请先完善个人信息，这样才能让广告主更好地了解您" → 强制跳转个人信息补充流程
- 超过接单数量：提示"每位创作者只能同时进行接单3个任务，确认接单吗？"
- 曾被拒绝：提示"您曾对该订单投标被拒绝，无法再次接单"
- 订单已过期：提示"该订单已过期"

#### 视频审核拒绝 ⭐ 重要
**前置条件**：广告主在投标管理中审核视频
**拒绝操作**：
- 广告主可以选择"修改"或"订单冻结"，不直接"拒绝"
- 但若广告主多次要求修改后仍不满意，或创作者严重违反要求，可标记为"审核拒绝"

**拒绝后的影响**：
- 投标状态更新为 `rejected`（已拒绝）
- 投标记录的 `isRejected` 字段标记为 `true`
- **该创作者永久无法再对该订单接单**
- 创作者工作台显示状态为"审核未通过"
- 创作者可以查看拒绝原因，但无法重新上传视频
- 订单的"已收到投稿数"保持不变（因为只是取消了这个创作者的投标）

**与其他创作者的关系**：
- 一个订单可以有多个创作者投标（采购数量）
- 某个创作者被拒绝，**不影响**其他创作者对该订单的投标
- 订单仍可在任务大厅展示，等待其他创作者接单

#### 视频验收通过
**前置条件**：投标状态为"预审已通过"、账户余额充足
**后置条件**：扣除尾款、生成授权书、开放无水印视频下载
