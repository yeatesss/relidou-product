description: 自动审查 Git 暂存区的代码变更，并根据 Conventional Commits 1.0.0 规范生成中文提交信息
---

请按照以下步骤生成 Git 提交信息：

## 1. 检查暂存区状态

首先检查当前暂存区的文件状态：

```bash
git status --short
git diff --cached --stat
```

如果暂存区为空，提示用户先使用 `git add` 暂存变更。

## 2. 审查变更详情

获取暂存区的详细变更内容：

```bash
git diff --cached
```

## 3. 分析变更并生成提交信息

根据变更内容，确定：

### 提交类型（Type）
- `feat` - 新增功能
- `fix` - 修复 bug
- `docs` - 文档修改
- `style` - 代码格式调整（不影响逻辑）
- `refactor` - 重构（不增加功能，不修复bug）
- `perf` - 性能优化
- `test` - 测试相关
- `build` - 构建系统或外部依赖变更
- `ci` - CI 配置文件和脚本变更
- `chore` - 其他不修改 src 或 test 的变更
- `revert` - 回滚之前的提交

### 作用域（Scope，可选）
描述代码库的某个部分，例如：`auth`、`api`、`ui` 等

### 描述（Description）
- 使用中文
- 简短总结代码变更
- 使用祈使句，现在时态
- 结尾不加句号
- 不超过 50 个字符

### 破坏性变更（Breaking Changes，可选）
如果包含破坏性变更，在类型后加 `!` 或在页脚中使用 `BREAKING CHANGE:`

## 4. 输出格式

请按以下格式输出：

```markdown
## 📋 暂存区变更总结

[列出修改的文件及主要变更]

## 🎯 建议的提交信息

```
<type>[optional scope]: <中文描述>

[可选正文]

[可选页脚]
```

## 💡 说明

[解释提交类型、作用域等选择理由]

## ✅ 提交命令

git commit -m "<type>[optional scope]: <中文描述>"
```

## 重要提醒

- **类型和作用域保持英文小写**
- **描述与正文使用中文**
- **使用祈使句**：例如"新增"而不是"新增了"
- **一次提交只做一件事**
- **先审查再提交**：确保暂存的变更都是相关的
